Classes e Objetos: Introdução à Programação Orientada a Objetos (POO)

Até agora, focamos na **programação procedural**, onde organizamos o código em funções que operam sobre dados. Embora eficaz para programas menores, essa abordagem pode se tornar complexa e difícil de manter em projetos maiores, pois os dados e as funções que os manipulam ficam separados.

A **Programação Orientada a Objetos (POO)** é um paradigma de programação que revoluciona essa organização. Em vez de focar em "o que fazer" (funções) e "com quais dados" (variáveis), a POO foca em "quem faz o quê". Ela permite que você modele o mundo real em seu código, combinando dados e as funções que operam nesses dados em unidades chamadas **objetos**.

As principais vantagens da POO incluem:

  • **Reusabilidade de Código:** Crie "moldes" que podem ser usados várias vezes.
  • **Modularidade:** Organize seu código em módulos autocontidos e independentes.
  • **Manutenibilidade:** Códigos bem estruturados em POO são mais fáceis de entender, depurar e modificar.
  • **Modelagem do Mundo Real:** Permite representar conceitos e entidades do mundo real de forma mais intuitiva no código.

Para entender POO, vamos focar em quatro conceitos-chave:

  • **Classe:** O molde ou a planta.
  • **Objeto (Instância):** Algo concreto criado a partir do molde.
  • **Atributo:** Uma característica ou dado que um objeto possui.
  • **Método:** Uma ação ou comportamento que um objeto pode realizar.

1. O que são Classes?

Uma **classe** é como um **molde, um protótipo, uma planta ou uma categoria** para criar objetos. Ela define a estrutura (quais dados os objetos terão) e o comportamento (quais ações os objetos poderão realizar) que todos os objetos criados a partir dessa classe terão.

Uma classe, por si só, não é um dado. Ela é apenas a **definição** de como os objetos serão criados. Por exemplo, a planta de um carro não é um carro, mas ela descreve como um carro deve ser construído.

Sintaxe Básica de uma Classe:

class NomeDaClasse:
    # Atributos de classe (opcional, compartilhados por todas as instâncias)
    # Métodos (funções que operam sobre os objetos desta classe)
    pass # 'pass' é usado quando o corpo da classe está vazio

Exemplo 1: Uma Classe Simples Carro

class Carro:
    # Esta classe ainda não tem atributos ou métodos específicos,
    # mas já serve como um molde básico.
    pass

print("Classe Carro criada com sucesso!")

2. O que são Objetos (Instâncias)?

Um **objeto** é uma **instância concreta** de uma classe. Se a classe é a planta, o objeto é o edifício construído a partir dessa planta. Você pode criar múltiplos objetos a partir da mesma classe, e cada objeto terá suas próprias características e estados, mas todos seguirão a estrutura definida pela classe.

Criando um Objeto (Instanciando uma Classe):

# Usando a classe Carro que definimos acima
meu_carro = Carro()    # Criamos um objeto 'meu_carro' da classe Carro
outro_carro = Carro()  # Criamos um segundo objeto 'outro_carro'

print(f"Tipo de 'meu_carro': {type(meu_carro)}")
print(f"Tipo de 'outro_carro': {type(outro_carro)}")

print(f"São o mesmo objeto? {meu_carro is outro_carro}") # Irá imprimir False

No exemplo acima, meu_carro e outro_carro são dois objetos distintos, mas ambos são do tipo Carro.

3. Atributos (Características dos Objetos)

**Atributos** são as **variáveis** que pertencem a um objeto. Eles armazenam os dados ou o estado do objeto. Cada objeto de uma classe terá seus próprios valores para esses atributos.

3.1. Atributos de Instância e o Método __init__

A maioria dos atributos que você usará são **atributos de instância**. Eles são definidos dentro do método especial __init__, que é o **construtor** da classe. O método __init__ é chamado automaticamente sempre que você cria um novo objeto da classe.

  • O primeiro parâmetro de __init__ (e de todos os métodos de instância) é sempre self. Ele representa a própria instância do objeto que está sendo criada.
  • Dentro de __init__, você usa self.nome_do_atributo = valor para criar e inicializar os atributos específicos daquele objeto.

Exemplo 3: Classe Carro com Atributos

class Carro:
    def __init__(self, marca, modelo, cor, ano):
        # Atributos de instância
        self.marca = marca
        self.modelo = modelo
        self.cor = cor
        self.ano = ano
        self.velocidade = 0 # Um atributo com valor inicial padrão

# Criando objetos (instâncias) da classe Carro
carro1 = Carro("Toyota", "Corolla", "Prata", 2022)
carro2 = Carro("Honda", "Civic", "Preto", 2023)
carro3 = Carro("Ford", "Fiesta", "Azul", 2018)

# Acessando os atributos dos objetos
print(f"Carro 1: {carro1.marca} {carro1.modelo} ({carro1.cor}, {carro1.ano})")
print(f"Velocidade do Carro 1: {carro1.velocidade} km/h")

print(f"Carro 2: {carro2.marca} {carro2.modelo} ({carro2.cor}, {carro2.ano})")
print(f"Velocidade do Carro 2: {carro2.velocidade} km/h")

# Podemos modificar atributos de um objeto específico
carro1.velocidade = 80
print(f"Nova velocidade do Carro 1: {carro1.velocidade} km/h")
print(f"Velocidade do Carro 2 (inalterada): {carro2.velocidade} km/h")

3.2. Atributos de Classe (Opcional)

Atributos de classe são definidos diretamente na classe (fora de qualquer método) e são **compartilhados** por todas as instâncias dessa classe. Eles são úteis para valores que são os mesmos para todos os objetos da classe.

Exemplo 4: Atributo de Classe rodas

class Carro:
    rodas = 4 # Atributo de CLASSE: todas as instâncias de Carro terão 4 rodas

    def __init__(self, marca, modelo):
        self.marca = marca
        self.modelo = modelo

carro_eletrico = Carro("Tesla", "Model 3")
carro_esportivo = Carro("Ferrari", "488")

print(f"Rodas do carro elétrico: {carro_eletrico.rodas}")
print(f"Rodas do carro esportivo: {carro_esportivo.rodas}")

# Você também pode acessar o atributo de classe diretamente pela classe
print(f"Rodas padrão para todos os Carros: {Carro.rodas}")

4. Métodos (Comportamentos dos Objetos)

**Métodos** são as **funções** que pertencem a uma classe e definem o que os objetos dessa classe podem fazer. Eles operam sobre os atributos do objeto. Assim como as funções normais, métodos podem receber argumentos.

  • O primeiro parâmetro de um método de instância deve ser sempre self, que se refere ao próprio objeto (instância) que está chamando o método.

Sintaxe de um Método:

class MinhaClasse:
    def meu_metodo(self, arg1, arg2):
        # Lógica do método, que pode usar self.atributo
        print(f"Executando meu_metodo com {arg1} e {arg2}")

Exemplo 5: Classe Carro com Métodos

class Carro:
    def __init__(self, marca, modelo):
        self.marca = marca
        self.modelo = modelo
        self.ligado = False # Estado inicial do carro

    def ligar(self):
        """Liga o carro se não estiver ligado."""
        if not self.ligado:
            self.ligado = True
            print(f"{self.marca} {self.modelo} foi ligado.")
        else:
            print(f"{self.marca} {self.modelo} já está ligado.")

    def desligar(self):
        """Desliga o carro se estiver ligado."""
        if self.ligado:
            self.ligado = False
            print(f"{self.marca} {self.modelo} foi desligado.")
        else:
            print(f"{self.marca} {self.modelo} já está desligado.")

    def acelerar(self, velocidade_aumento):
        """Aumenta a velocidade do carro."""
        if self.ligado:
            print(f"{self.marca} {self.modelo} está acelerando.")
        else:
            print(f"Não é possível acelerar! {self.marca} {self.modelo} está desligado.")

# Criando um objeto Carro
meu_carro_sonho = Carro("Porsche", "911")

# Chamando os métodos do objeto
meu_carro_sonho.ligar()
meu_carro_sonho.acelerar(50)
meu_carro_sonho.desligar()
meu_carro_sonho.desligar() # Tentando desligar novamente
meu_carro_sonho.acelerar(20) # Tentando acelerar desligado

5. Encapsulamento Básico (Convenções de Nomenclatura)

Encapsulamento é um dos princípios da POO que sugere que os detalhes internos de um objeto devem ser "escondidos" do mundo exterior. Isso significa que o acesso aos dados internos de um objeto deve ser feito através de seus métodos, e não diretamente.

Em Python, o encapsulamento é mais uma **convenção** do que uma imposição rigorosa, como em outras linguagens (Java, C++).

  • **Atributos/Métodos "Protegidos" (_nome):** Se um atributo ou método começa com um **único underscore** (ex: _saldo), é uma convenção para indicar que ele é "protegido" ou "interno". Você ainda pode acessá-lo diretamente, mas a ideia é que ele seja usado apenas pela própria classe ou suas subclasses.
  • **Atributos/Métodos "Privados" (__nome):** Se começa com **dois underscores** (ex: __saldo), o Python realiza um processo chamado "name mangling" (distorção de nomes). Isso torna o acesso direto um pouco mais difícil (mas não impossível), como _NomeDaClasse__nome. É um indicativo mais forte de que o atributo/método é privado.

Exemplo 6: Encapsulamento Básico com Getters e Setters

class ContaBancaria:
    def __init__(self, saldo_inicial):
        # Usamos '__saldo' para indicar que é um atributo que não deve ser acessado diretamente
        self.__saldo = saldo_inicial 

    def depositar(self, valor):
        """Deposita um valor na conta, com validação."""
        if valor > 0:
            self.__saldo += valor
            print(f"Depósito de R${valor:.2f} realizado. Novo saldo: R${self.__saldo:.2f}")
        else:
            print("Valor de depósito inválido. Deve ser positivo.")

    def sacar(self, valor):
        """Saca um valor da conta, com validação."""
        if valor > 0 and valor <= self.__saldo:
            self.__saldo -= valor
            print(f"Saque de R${valor:.2f} realizado. Novo saldo: R${self.__saldo:.2f}")
        elif valor <= 0:
            print("Valor de saque inválido. Deve ser positivo.")
        else:
            print("Saldo insuficiente para o saque.")

    def get_saldo(self): # Método "getter" para acessar o saldo de forma controlada
        """Retorna o saldo atual da conta."""
        return self.__saldo

# Criando um objeto ContaBancaria
minha_conta = ContaBancaria(1000)

print(f"Saldo inicial: R${minha_conta.get_saldo():.2f}")

minha_conta.depositar(200)
minha_conta.sacar(150)
minha_conta.sacar(2000) # Tentando sacar mais do que tem
minha_conta.depositar(-50) # Tentando depositar valor inválido

# Tentando acessar o atributo "privado" diretamente (Python não impede, mas desaconselha)
# print(minha_conta.__saldo) # Isso causaria um AttributeError: 'ContaBancaria' object has no attribute '__saldo'
# Você teria que usar: print(minha_conta._ContaBancaria__saldo) para acessá-lo.

Apesar de Python não ter um encapsulamento forte, a prática de usar métodos (como get_saldo(), depositar(), sacar()) para interagir com os atributos internos é uma boa maneira de garantir que a lógica de validação seja aplicada e que o objeto permaneça em um estado consistente.

Conclusão

A Programação Orientada a Objetos (POO) é um conceito poderoso que permite que você organize seu código de forma mais intuitiva e eficiente, modelando entidades do mundo real como objetos com características (atributos) e comportamentos (métodos).

Dominar os conceitos de **Classe**, **Objeto**, **Atributo** e **Método** é o primeiro passo crucial para desbloquear todo o potencial da POO em Python. Nos próximos módulos, exploraremos outros pilares da POO, como Herança, Polimorfismo e Abstração, que aprofundam ainda mais essa capacidade de modelagem.

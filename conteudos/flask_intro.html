Introdução ao Desenvolvimento Web com Flask

Você já aprendeu a criar scripts Python que realizam tarefas úteis, como coletar dados de APIs ou páginas web. Mas e se você quiser criar uma **interface para seus usuários**, onde eles possam interagir com seu código através de um navegador web? É aí que entra o **desenvolvimento web**.

O desenvolvimento web é dividido em duas partes principais:

  • Frontend: Tudo o que o usuário vê e interage no navegador (HTML, CSS, JavaScript).
  • Backend: O servidor, o banco de dados e a lógica de negócio por trás da aplicação. É aqui que seu código Python entra.

Para construir a parte do backend de uma aplicação web com Python, usamos **Frameworks Web**. Um framework é um conjunto de ferramentas, bibliotecas e convenções que simplificam o processo de desenvolvimento, lidando com tarefas comuns como roteamento de URLs, tratamento de requisições HTTP e renderização de páginas.

Existem vários frameworks web Python, como Django e Pyramid. Nesta seção, vamos nos concentrar no **Flask**, um **microframework** popular e excelente para começar a aprender desenvolvimento web. Ele é leve, flexível e permite que você comece a criar aplicações rapidamente, sem muita complexidade inicial.

1. O que é Flask?

Flask é um framework web minimalista e flexível para Python. A palavra "micro" em microframework significa que ele não inclui ferramentas como um ORM (Object-Relational Mapper) para bancos de dados ou bibliotecas para formulários e autenticação por padrão. Em vez disso, ele oferece os recursos essenciais e permite que o desenvolvedor escolha as extensões e bibliotecas que melhor se adequam ao seu projeto.

Isso o torna uma ótima escolha para:

  • Construir APIs RESTful.
  • Desenvolver protótipos rápidos.
  • Criar aplicações pequenas a médias que precisam de muita personalização.
  • Aprender os fundamentos do desenvolvimento web sem sobrecarga.

Seus principais componentes incluem um servidor de desenvolvimento embutido, um roteador (para mapear URLs para funções Python) e um sistema de templates (Jinja2) para gerar HTML dinamicamente.

2. Instalando o Flask

Como o Flask é uma biblioteca de terceiros, você precisará instalá-lo usando pip. Lembre-se de sempre fazer isso dentro de um ambiente virtual para o seu projeto Flask!

# Ative seu ambiente virtual primeiro!
pip install Flask

3. Sua Primeira Aplicação Flask: "Hello, World!"

Vamos criar nosso primeiro aplicativo Flask. Ele será simples: uma página que exibe "Olá, Mundo!" e outra página "Sobre".

Crie um arquivo chamado app.py na raiz do seu projeto e cole o código abaixo:

Arquivo: app.py

# app.py
from flask import Flask

# Cria uma instância da aplicação Flask.
# __name__ é uma variável especial do Python que contém o nome do módulo atual.
# Flask a usa para determinar a raiz da aplicação.
app = Flask(__name__)

# Define uma rota (route) para a URL raiz ('/') da sua aplicação.
# Quando um usuário acessa essa URL, a função 'hello_world' será executada.
@app.route('/')
def hello_world():
    # Retorna uma string HTML que será exibida no navegador.
    return '<h1>Olá, Mundo com Flask!</h1>'

# Define outra rota para a URL '/sobre'.
@app.route('/sobre')
def about():
    return '<p>Esta é uma página simples sobre o Flask.</p>'

# Este bloco verifica se o script está sendo executado diretamente (e não importado).
# É o ponto de entrada para rodar a aplicação Flask.
if __name__ == '__main__':
    # app.run() inicia o servidor de desenvolvimento do Flask.
    # debug=True: Ativa o modo de depuração. Isso recarrega o servidor automaticamente
    # a cada alteração no código e fornece informações de erro detalhadas no navegador.
    # Use 'debug=False' em produção!
    app.run(debug=True)

Como Rodar sua Aplicação:

  1. Salve o código acima como app.py em uma pasta.
  2. Abra seu terminal ou prompt de comando.
  3. Navegue até a pasta onde você salvou app.py.
  4. Certifique-se de que seu ambiente virtual esteja ativado (onde você instalou o Flask).
  5. Execute o script: 
    python app.py
  6. Você verá uma saída no terminal indicando que o servidor está rodando, geralmente em http://127.0.0.1:5000/.
  7. Abra seu navegador e acesse:
    • http://127.0.0.1:5000/ para ver "Olá, Mundo com Flask!".
    • http://127.0.0.1:5000/sobre para ver a página "Sobre".

4. Rotas Dinâmicas e Variáveis na URL

Você pode tornar suas URLs mais dinâmicas, capturando partes delas como variáveis que serão passadas para sua função de view. Isso é útil para exibir informações específicas com base na URL (ex: perfil de um usuário, detalhes de um produto).

Você pode definir o tipo da variável na rota (ex: <int:id> para um número inteiro, <string:nome> para texto). Se o tipo não for especificado, ele assume `string` por padrão.

Exemplo 4.1: Rotas com Variáveis (Adicione ao seu app.py)

# app.py
from flask import Flask

app = Flask(__name__)

@app.route('/')
def hello_world():
    return '<h1>Olá, Mundo com Flask!</h1>'

@app.route('/sobre')
def about():
    return '<p>Esta é uma página simples sobre o Flask.</p>'

# Rota dinâmica para saudar um nome
@app.route('/saudacao/<nome>')
def saudacao(nome):
    return f'<h1>Olá, {nome.capitalize()}!</h1>'

# Rota dinâmica para calcular a soma de dois números inteiros
@app.route('/calcula/<int:num1>/<int:num2>')
def calcula_soma(num1, num2):
    soma = num1 + num2
    return f'<h1>A soma de {num1} e {num2} é: {soma}</h1>'

if __name__ == '__main__':
    app.run(debug=True)

Reinicie o `app.py` no terminal e acesse no navegador:

  • http://127.0.0.1:5000/saudacao/daniel
  • http://127.0.0.1:5000/calcula/10/5

5. Renderizando Templates (HTML Externo)

Retornar strings HTML diretamente das funções de view é bom para exemplos simples, mas não é prático para páginas complexas. É por isso que frameworks web usam **templates**. Templates são arquivos (geralmente HTML) que contêm marcadores de posição para dados dinâmicos e lógica de controle.

O Flask usa o motor de templates **Jinja2**. Você pode passar variáveis do seu código Python para o template, e o Jinja2 as preencherá dinamicamente.

Estrutura de Pastas para Templates:

Por padrão, o Flask procura por templates em uma pasta chamada templates na raiz do seu projeto. Crie essa pasta:

seu_projeto_flask/
├── app.py
└── templates/
    ├── index.html
    └── sobre.html

Crie os Arquivos de Template:

Arquivo: `templates/index.html`

<!DOCTYPE html>
<html lang="pt-br">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>{{ titulo }}</title> <!-- Variável Python injetada aqui -->
    <!-- Link para o arquivo CSS estático (ver próxima seção) -->
    <link rel="stylesheet" href="{{ url_for('static', filename='css/style.css') }}">
</head>
<body>
    <h1>Bem-vindo à {{ titulo }}!</h1>
    <p>Este é um parágrafo da página renderizado por um template.</p>
    <p>A mensagem do servidor é: <strong>{{ mensagem }}</strong></p>
    <p>Seu nome (se houver): <strong>{{ nome_usuario if nome_usuario else 'Visitante' }}</strong></p>
    
    <ul>
        <li><a href="/sobre">Sobre nós</a></li>
        <li><a href="/contato">Página de Contato</a></li>
        <li><a href="/saudacao/Visitante">Saudar Visitante</a></li>
    </ul>

    <!-- Exibir mensagens flash (se houver) -->
    <div class="flash-messages">
        {% with messages = get_flashed_messages(with_categories=true) %}
            {% if messages %}
                <ul>
                {% for category, message in messages %}
                    <li class="{{ category }}">{{ message }}</li>
                {% endfor %}
                </ul>
            {% endif %}
        {% endwith %}
    </div>

</body>
</html>

Arquivo: `templates/sobre.html`

Modifique seu app.py para Usar Templates:

Importe render_template e altere as funções de view:

app = Flask(__name__)

@app.route('/')
def home():
    # render_template busca o arquivo na pasta 'templates'
    # Você pode passar quantas variáveis nomeadas quiser para o template

                           titulo='Minha Aplicação Web', 
                           mensagem='Dados do backend injetados aqui!', 
                           nome_usuario='Aprendiz de Flask')

@app.route('/sobre')
def about_template():
    return render_template('sobre.html') # Pode passar variáveis aqui também, se precisar

# ... (Mantenha as rotas /saudacao/<nome> e /calcula/<int:num1>/<int:num2> do exemplo anterior) ...

if __name__ == '__main__':
    app.run(debug=True)

Reinicie o servidor e acesse http://127.0.0.1:5000/. Você verá o conteúdo do `index.html` com as variáveis Python preenchidas. O Jinja2 usa {{ variavel }} para exibir variáveis e {% control %} para lógica (como loops ou condicionais).

6. Servindo Arquivos Estáticos (CSS, JS, Imagens)

7. Recebendo Dados de Formulários (GET e POST)

A interação mais comum em aplicações web é através de formulários. O Flask facilita a captura de dados enviados por eles.

Para lidar com formulários, você usará o objeto `request` do Flask, que contém todos os dados da requisição HTTP (método, argumentos de URL, dados de formulário, etc.).

Arquivo: `templates/formulario.html`

<!DOCTYPE html>
<html lang="pt-br">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Formulário de Contato</title>
    <link rel="stylesheet" href="{{ url_for('static', filename='css/style.css') }}">
</head>
<body>
    <h1>Envie sua Mensagem</h1>
    <form action="/enviar_mensagem" method="POST">
        <label for="nome">Nome:</label><br>
        <input type="text" id="nome" name="nome" required><br><br>
        
        <label for="email">Email:</label><br>
        <input type="email" id="email" name="email" required><br><br>
        
        <label for="mensagem">Mensagem:</label><br>
        <textarea id="mensagem" name="mensagem" rows="5" cols="30" required></textarea><br><br>
        
        <input type="submit" value="Enviar Mensagem">
    </form>
    <p><a href="/">Voltar para a Home</a></p>
</body>
</html>

Modifique seu app.py para Lidar com Formulários:

# app.py
from flask import Flask, render_template, request, redirect, url_for, flash

app = Flask(__name__)
# A secret_key é necessária para usar flash messages (mensagens temporárias)
# Em produção, use uma chave forte e a mantenha secreta!
app.secret_key = 'sua_chave_secreta_aqui_para_flash_messages_muito_segura' 

@app.route('/')
def home():
    # Mensagens flash são automaticamente injetadas nos templates através de get_flashed_messages()
    return render_template('index.html', 
                           titulo='Minha Aplicação Web', 
                           mensagem='Dados do backend injetados aqui!', 
                           nome_usuario='Aprendiz de Flask')

@app.route('/sobre')
def about_template():
    return render_template('sobre.html')

# ... (Mantenha as rotas /saudacao/<nome> e /calcula/<int:num1>/<int:num2> do exemplo anterior) ...

# Rota para exibir o formulário de contato (método GET padrão)
@app.route('/contato')
def contato_form():
    return render_template('formulario.html')

# Rota para processar os dados do formulário (apenas aceita método POST)
@app.route('/enviar_mensagem', methods=['POST'])
def enviar_mensagem():
    # Verifica se a requisição foi feita com o método POST
    if request.method == 'POST':
        # request.form é um dicionário contendo os dados do formulário enviados via POST
        nome = request.form['nome']
        email = request.form['email']
        mensagem = request.form['mensagem']
        
        print(f"--- Mensagem Recebida via Formulário ---")
        print(f"Nome: {nome}")
        print(f"Email: {email}")
        print(f"Mensagem: {mensagem}")
        print(f"----------------------------------------")

        # Exibe uma mensagem temporária para o usuário
        flash(f'Obrigado, {nome}! Sua mensagem foi enviada com sucesso.', 'success')
        
        # Redireciona o usuário para a página inicial após o envio
        # url_for('home') gera a URL para a função 'home'
        return redirect(url_for('home'))
    
    # Se a requisição não for POST (ex: alguém tentar acessar /enviar_mensagem via GET),
    # retorna um erro 405 (Método Não Permitido)
    return "Método não permitido", 405

if __name__ == '__main__':
    app.run(debug=True)

Reinicie o `app.py`. Acesse http://127.0.0.1:5000/contato, preencha o formulário e envie. Você verá a mensagem no seu terminal e será redirecionado para a home com uma mensagem de sucesso.

Observe que usamos `app.secret_key` e `flash()`/`get_flashed_messages()`. As mensagens flash são uma forma de passar mensagens temporárias do backend para o frontend, úteis para feedback ao usuário (sucesso, erro, aviso).

8. Boas Práticas e Próximos Passos

  • Modo Debug: Sempre desative debug=True em ambientes de produção. Ele expõe informações sensíveis e pode levar a vulnerabilidades.
  • Estrutura de Projeto: Para projetos maiores, organize seu Flask em pacotes e módulos, em vez de um único `app.py`. Isso é chamado de "estruturação de blueprints".
  • Extensões Flask: O ecossistema Flask é rico em extensões que adicionam funcionalidades (ex: Flask-SQLAlchemy para banco de dados, Flask-Login para autenticação, Flask-WTF para formulários mais seguros).
  • Jinja2: Explore mais o Jinja2! Ele permite loops (`{% for item in lista %}`), condicionais (`{% if condicao %}`), herança de templates (`{% extends 'base.html' %}`), e muito mais para criar HTML dinâmico.
  • Outros Frameworks: O Django é um framework "batteries-included" (com tudo embutido) mais robusto, ideal para aplicações grandes e complexas. Após dominar o Flask, pode ser uma próxima etapa interessante.

Exercício 1: Criar um Contador de Cliques Simples

Crie uma aplicação Flask com uma página que exiba um número. Cada vez que o usuário recarrega a página ou clica em um botão "Incrementar", o número deve aumentar em 1. O contador deve ser mantido na memória do servidor enquanto a aplicação estiver rodando.

Dicas:

  • Declare uma variável global (ex: `contador = 0`) no seu app.py (fora de qualquer função de rota).
  • Na sua função de rota, incremente essa variável.
  • Passe a variável `contador` para o seu template HTML.
  • No HTML, crie um botão que seja um link para a mesma URL da página, o que fará com que a página seja recarregada e a função de rota seja executada novamente.
# Exemplo de app.py para o Exercício 1
from flask import Flask, render_template

app = Flask(__name__)

# Variável global para o contador
contador_cliques = 0

@app.route('/contador')
def pagina_contador():
    global contador_cliques # Declare que você quer usar a variável global
    contador_cliques += 1 # Incrementa o contador
    return render_template('contador.html', contador=contador_cliques)

# if __name__ == '__main__':
#     app.run(debug=True)

# Lembre-se de criar o arquivo templates/contador.html:
# <!DOCTYPE html>
# <html lang="pt-br">
# <head>
#     <meta charset="UTF-8">
#     <title>Contador de Cliques</title>
#     <link rel="stylesheet" href="{{ url_for('static', filename='css/style.css') }}">
# </head>
# <body>
#     <h1>Contador de Cliques</h1>
#     <p>Você clicou <strong>{{ contador }}</strong> vezes.</p>
#     <p><a href="/contador"><button>Clique para Incrementar!</button></a></p>

Exercício 2: Lista de Tarefas (To-Do List) Básica

Crie uma aplicação Flask com as seguintes funcionalidades:

  • Uma página principal que exiba uma lista de tarefas (To-Do List).
  • Um formulário na mesma página para adicionar novas tarefas à lista.
  • As tarefas devem ser armazenadas em uma lista Python na memória do servidor (serão perdidas ao reiniciar o servidor).

Dicas:

  • Use uma lista Python global (ex: `tarefas = []`).
  • A rota principal (`/`) deve lidar com requisições GET (para exibir a lista e o formulário) e POST (para adicionar a nova tarefa).
  • No formulário HTML, use `method="POST"` e um campo de texto para a tarefa.
  • Após adicionar uma tarefa, use `redirect(url_for('nome_da_rota_principal'))` para evitar que a tarefa seja adicionada novamente se o usuário recarregar a página.
# Exemplo de app.py para o Exercício 2
from flask import Flask, render_template, request, redirect, url_for

app = Flask(__name__)
app.secret_key = 'chave_secreta_para_todo_list'

# Lista global para armazenar as tarefas (na memória)
tarefas = []

@app.route('/todo', methods=['GET', 'POST'])
def todo_list():
    if request.method == 'POST':
        nova_tarefa = request.form.get('tarefa')
        if nova_tarefa:
            tarefas.append(nova_tarefa)
        return redirect(url_for('todo_list')) # Redireciona para evitar re-submissão
    
    return render_template('todo.html', tarefas=tarefas)

# if __name__ == '__main__':
#     app.run(debug=True)

# Lembre-se de criar o arquivo templates/todo.html:
